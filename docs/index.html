playground

A TypeScript scratch space with no build step.

Node runs .ts files directly via type stripping — no bundler, no tsx, no compile output. TypeScript is here to type-check, not to build.

Try it

git clone https://github.com/micrubdev/playground.git
cd playground
npm install
npm start        # runs src/index.ts directly

What's inside

Vitest for tests, type-aware ESLint, Prettier, a Husky pre-commit hook, and CI — all gated behind one npm run check. The tradeoffs behind each choice are written up in CLAUDE.md.
